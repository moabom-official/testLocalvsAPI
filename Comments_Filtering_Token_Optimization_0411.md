# Tech Product Comment Analysis Pipeline (중복 기반 다중 기준 선별 구조)

# 이모지
이모지

## 개요
본 시스템은 YouTube 테크 제품 리뷰 댓글을 분석하기 위해  
**다중 기준 추출 + 중복 등장 기반 선별 방식**을 사용한다.

가중치 기반 점수식 대신:

- 여러 기준으로 독립적으로 댓글을 추출하고
- 여러 기준에서 반복적으로 등장한 댓글을 우선 선택하며
- 단일 기준 댓글은 보조 점수 기반으로 선별한다

---

## 핵심 원칙

1. **하드 필터링 없음**
   - 키워드가 없다고 댓글을 제거하지 않는다

2. **다중 기준 독립 추출**
   - 여러 기준으로 댓글을 각각 선별

3. **중복 등장 = 높은 중요도**
   - 여러 기준에서 선택된 댓글은 중요도가 높다고 판단

4. **단일 등장 댓글 = 보조 후보**
   - 보조 점수 기준으로 선별

5. **토큰 최적화**
   - LLM에 전달되는 댓글 수를 제한

---

## 전체 파이프라인 구조

```text
[상품 등록]
    ↓
카테고리 선택
브랜드 입력
제품명 입력
    ↓
키워드 프로필 생성
    ↓
[댓글 수집]
YouTube 댓글 수집
초기 후보군 구성 (예: 100개)
    ↓
[전처리]
정규화
중복 제거
    ↓
[다중 기준 후보 추출]
각 기준별 상위 댓글 선택:
- 좋아요 상위
- 답글 수 상위
- 길이 상위
- 키워드 포함
- 최신 댓글
- 랜덤 샘플
    ↓
[중복 등장 계산]
각 댓글에 대해:
- 등장 횟수(hit_count)
- 등장 기준(sources)
    ↓
[1차 선별]
hit_count ≥ 2 댓글 선택
    ↓
[2차 선별]
부족한 경우:
hit_count = 1 댓글 중에서 선별
    ↓
[최종 후보 구성]
목표 개수 (예: 20개)로 제한
    ↓
[토큰 예산 검사]
- 일일 토큰 제한: 60000
- 영상별 토큰 제한
- 배치 토큰 제한
- 댓글 길이 제한
    ↓
[LLM 분류]
배치 단위 처리
    ↓
[Agent 판단]
ANALYZE / EXCLUDE
    ↓
[Aspect 분석]
PRODUCT_OPINION 댓글만 추가 분석
    ↓
DB 저장 및 보고서 생성
````

## 다중 기준 후보 추출

```text
- 좋아요 상위 댓글("like")
- 답글 많은 댓글("many")
- 길이가 긴 댓글("long")
- 최신 댓글("new")
- 오래된 댓글("old")
```

---

## 초기 댓글 후보군 구성 (1차 선별 이전 단계)

댓글 수집 단계에서는 단순히 최신 댓글이나 인기 댓글만 사용하는 것이 아니라,  
대표성을 확보하기 위해 다양한 기준을 기반으로 초기 후보군을 구성한다.

---

## 목적

```text
다양한 관점(인기, 최신, 논쟁, 정보성)을 반영한 균형 잡힌 댓글 집합 생성
```

## 불러오는 댓글 종류
```
- 좋아요 상위 댓글(20개)
- 답글 많은 댓글(20개)
- 길이가 긴 댓글(20개)
- 최신 댓글(20개)
- 오래된 댓글(20개)
```


## 중복 등장 계산

```python
comment_meta = {
    "comment_id": "...",
    "hit_count": 2,
    "sources": ["like", "keyword"]
}
```

---

## 1차 선별

```text
조건:
hit_count ≥ 2
```

---

## 2차 선별 (단일 등장 댓글 처리)

### 보조 점수 계산

단일 등장 댓글에 대해 간단한 보조 점수 계산:

```python
secondary_score =
    normalized_like +
    normalized_reply +
    keyword_hit_count
```

---

### 정렬 기준

```text
1순위: secondary_score (내림차순)
2순위: 댓글 길이 (내림차순)
```

---

### 선택 방식

```text
hit_count = 1 댓글을 위 기준으로 정렬 후
상위부터 남은 슬롯만큼 선택
```

---

## 최종 구성 예시

```text
총 20개 기준:

- hit_count ≥ 2 → 12개
- hit_count = 1 → 8개 (보조 점수 기준)
```
