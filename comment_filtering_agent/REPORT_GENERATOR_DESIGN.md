# 보고서 생성기 설계 문서

## 개요

댓글 분석 파이프라인의 최종 단계로, 정제된 데이터를 바탕으로 실무에서 활용 가능한 제품 인사이트 보고서를 생성합니다.

---

## 핵심 설계 원칙

### 1. **Signal vs Noise 분리**

```
전체 댓글 1,247개
    ↓
[필터링]
    ↓
Signal (687개) ← 이것만 분석
- PRODUCT_OPINION: 제품 평가
- QUESTION: 진성 관심사

Noise (560개) ← 제외
- VIDEO_REACTION: "영상 재밌어요"
- CHATTER: "ㅋㅋㅋ"
- OFF_TOPIC: "배경음악 뭐예요?"
- SPAM/ABUSIVE: 스팸, 욕설
```

**왜 중요한가?**
- 잡담이 섞이면 통계가 왜곡됨
- "구독했어요"를 긍정 의견으로 집계하면 안 됨
- 실제 제품 평가만 필터링해야 신뢰도 높은 인사이트 도출

```python
{
  "total_collected": 1247,      # 수집된 전체 댓글
  "rule_filter_rejected": 423,  # 1차 필터 제외

  "exclusion_rate": 33.9,       # 제외율 (%)
  "analysis_rate": 55.1         # 분석 비율 (%)
```

### 3. 감정 분포
```python
{
  "positive_count": 412,
  "neutral_count": 189,
  "negative_count": 86,
  "sentiment_score": +42.3      # -100 ~ +100

```

**감정 스코어 해석**
- `+70 ~ +100`: 압도적 긍정
- `+30 ~ +70`: 긍정적
- `-30 ~ +30`: 중립/혼재
- `-70 ~ -30`: 부정적
- `-100 ~ -70`: 압도적 부정

### 4. Aspect 분석
```python

  "positive_count": 42,
```

**Aspect별 인사이트**
- 언급량: 사용자 관심도
- 감정 분포: 만족/불만 수준
- 스코어: 종합 평가

## 실무적으로 유의미한 지표

### 1. **감정 스코어 (Sentiment Score)**
```
공식: (긍정 - 부정) / 전체 * 100
범위: -100 ~ +100
```

**왜 중요한가?**
- 단일 숫자로 제품 평가 요약
- 경쟁사 비교 가능
- 시계열 트렌드 추적

**실무 활용**
- 마케팅: +50 이상이면 "고객 만족도 높음" 강조
- 제품팀: -20 이하면 개선 필요
- 경영진: 분기별 트렌드 보고

### 2. **Aspect별 감정 분포**
```
발열: -22 (77개 부정)
카메라: +73 (198개 긍정)
```

**왜 중요한가?**
- 제품의 강점/약점 명확히 파악
- 개선 우선순위 결정
- 마케팅 메시지 최적화

**실무 활용**
- 제품팀: 발열 개선 최우선 과제
- 마케팅: 카메라 강조, 발열 해명
- R&D: 다음 모델 기획

### 3. **질문 주제 분포**
```
게임: 87개 질문
가격: 52개 질문
카메라: 43개 질문
```

**왜 중요한가?**
- 구매 의향자의 진짜 관심사
- FAQ 우선순위
- 콘텐츠 기획 아이디어

**실무 활용**
- 콘텐츠팀: 게임 성능 상세 테스트 영상 제작
- CS팀: 게임 관련 FAQ 보강
- 마케팅: 게이머 타겟 캠페인

### 4. **제외율 (Exclusion Rate)**
```
제외율: 33.9% (560개 / 1,247개)
```

**왜 중요한가?**
- 필터링 품질 지표
- 데이터 신뢰도 지표

**해석**
- 30~40%: 정상 (잡담/스팸 적절히 제거)
- 10% 이하: 필터 약함 (노이즈 포함)
- 70% 이상: 필터 강함 (과도한 제외)

### 5. **분석 비율 (Analysis Rate)**
```
분석 비율: 55.1% (687개 / 1,247개)
```

**왜 중요한가?**
- 실제 제품 평가 댓글 비율
- 영상 품질 간접 지표

**해석**
- 50% 이상: 제품 중심 영상 (good)
- 30% 이하: 잡담/반응 많음 (영상 엔터테인먼트 중심)

### 6. **대표 댓글 좋아요 수**
```
"카메라 진짜 미쳤네요" - 342 likes
"가격이 너무 비싸요" - 412 likes
```

**왜 중요한가?**
- 다수의 공감을 받은 의견
- 실제 여론 반영

**실무 활용**
- 마케팅: 인용구로 활용 가능
- 제품팀: 우선 대응 필요
- 홍보팀: 긍정 댓글 확대 재생산

## 왜 잡담 필터링이 중요한가?

### 필터링 없는 분석의 문제점

#### 시나리오 1: 잡담이 긍정으로 오분류
```
"ㅋㅋㅋㅋ 재밌네요" → 긍정?
"구독했어요!!" → 긍정?
"1등" → 긍정?
```

**결과**: 감정 스코어 인플레이션
- 실제: +42.3 (신뢰)
- 필터 없이: +68.7 (과장)

#### 시나리오 2: 무관 댓글이 노이즈
```
"배경음악 제목 뭐예요?" → Aspect?
"첫 댓글!" → 의미?
"여기 광고 들어갑니다" → 스팸
```

**결과**: Aspect 통계 오염
- "배경음악"이 주요 관심사로 집계됨
- 실제 제품 특성 묻힘

#### 시나리오 3: 영상 반응 혼입
```
"오늘도 잘 봤습니다"
"편집 좋아요"
"목소리 좋으시네요"
```

**결과**: 제품 vs 영상 구분 불가
- 리뷰어 만족도 ≠ 제품 만족도

### 필터링 후 효과

#### 1. **통계 정확도 향상**
```
필터링 전: 긍정 812개 (잡담 포함)
필터링 후: 긍정 412개 (순수 제품 평가)

→ 2배 차이!
```

#### 2. **Aspect 신뢰도 향상**
```
필터링 전 Top 5:
1. "영상" (192개) ← 제품 아님
2. "카메라" (247개)
3. "구독" (134개) ← 제품 아님
4. "성능" (231개)
5. "재밌다" (98개) ← 제품 아님

필터링 후 Top 5:
1. 카메라 (247개)
2. 성능 (231개)
3. 디스플레이 (189개)
4. 발열 (156개)
5. 배터리 (143개)
```

**차이 명확!**

#### 3. **인사이트 품질 향상**
```
필터링 전 약점:
- "영상 길이" ← 제품 아님
- "발열"
- "배경음악" ← 제품 아님

필터링 후 약점:
- "발열"
- "가격"
- "소음"

→ 제품 개선 포인트 명확!
```

### Agent 기반 필터링의 가치

#### 전통적 방법 (키워드 기반)
```python
if "좋아요" in comment:
    sentiment = "positive"  # 위험!
```

**문제**:
- "배터리가 좋아요" → OK
- "영상이 좋아요" → 제품 아님
- "좋아요 누르고 갑니다" → 제품 아님

#### Agent 기반 방법
```
1차 필터: "좋아요 누르고 갑니다" → REJECT (LOW_INFORMATION)
2차 분류: "영상이 좋아요" → VIDEO_REACTION
Agent: VIDEO_REACTION → EXCLUDE

최종: "배터리가 좋아요"만 분석
```

**결과**: 정확도 향상

## 사용 예시

```python
from comment_filtering_agent.services.report_generator import (
    ReportGenerator, ReportConfig
)

# 설정
config = ReportConfig(
    top_aspects_count=10,
    representative_comments_count=5,
    output_dir="reports"
)

# 보고서 생성
generator = ReportGenerator(config)
report_data = generator.generate_report(
    video_id="abc123",

    video_title="갤럭시 S25 Ultra 리뷰",
    product_name="갤럭시 S25 Ultra"

# Markdown 저장
generator.save_markdown(report_data)
# → reports/report_abc123.md

# JSON 저장
generator.save_json(report_data)
# → reports/report_abc123.json

# 프로그래밍 방식 접근
print(f"감정 스코어: {report_data.overall_sentiment.sentiment_score}")
print(f"주요 강점: {report_data.insight.strengths[0]}")
print(f"주요 약점: {report_data.insight.weaknesses[0]}")
```

## 요약

✅ **필터링의 가치**
- 잡담 제거 → 통계 정확도 2배 향상
- 제품 평가만 분석 → 신뢰도 높은 인사이트
- Agent 기반 → 문맥 이해 분류

✅ **실무 지표**
- 감정 스코어: 제품 평가 요약
- Aspect 분석: 강점/약점 명확화
- 질문 주제: 사용자 관심사
- 제외율: 필터 품질 지표

✅ **출력 형식**
- Markdown: 사람이 읽기 쉬움
- JSON: API/대시보드 연동
- 구조화: BI 도구 활용

**핵심**: 잡담이 많아도 괜찮습니다. Agent가 정확히 걸러내기 때문에 최종 보고서는 순수한 제품 인사이트만 담깁니다! 🎯
